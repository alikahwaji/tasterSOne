Introduction to p5.js

p5.js is a visual-first JavaScript library designed specifically for artists, designers, educators, and beginners.

  • Simplifies drawing complex math shapes in browser.
  • Exposes simple commands for coordinates, colors, and motion.
  • Perfect sandbox for learning fundamental programming loops.

Why p5.js?

Traditional code starts with text outputs like "Hello World." With p5.js, we immediately start drawing, making logic bugs easy to see and correct!

The p5.js Web Editor

Open your browser and navigate to: editor.p5js.org

The editor is structured into three main zones:

  • Left Panel (Code Editor): Write your scripts and statements.
  • Right Panel (Preview Canvas): See shapes draw instantly.
  • Top Bar (Play & Stop): Run your loop or halt execution.

// Editor Preview Setup

function setup() {
  createCanvas(400, 400);
}
function draw() {
  background(220);
}

Setup vs. Draw Loops

Understanding these two core functions is crucial for p5.js animation.

  • `setup()`: Runs **exactly once** when the program loads. Use it to configure initial options like `createCanvas`.
  • `draw()`: Runs **continuously** at 60 frames per second (60 FPS) in a repeating loop. Think of it like pages flipping in an animation flipbook.

Why separate them?

Initialization (setting bounds, canvas sizing) only needs to happen once. Redrawing animations, moving items, and tracking mouse coordinates happen constantly in `draw()`!

Canvas Coordinates

Unlike school graphs where the origin `(0,0)` is in the center, screens start at the **top-left corner**.

  • Origin `(0, 0)` is the very top-left pixel.
  • X coordinate: Distance moving from left to right.
  • Y coordinate: Distance moving from top to bottom.
  • Grid boundaries extend to the `width` and `height` of the canvas.

Grid Positions

To draw shapes, you must tell p5.js the exact horizontal (X) and vertical (Y) coordinate values.

  • If canvas size is `createCanvas(600, 400)`:
  • The center point is `(300, 200)`.
  • The middle-left edge is `(0, 200)`.
  • The bottom-center edge is `(300, 400)`.

Quick Formula:

X = width / 2; // Center horizontal
Y = height / 2; // Center vertical

Drawing Circles: ellipse()

To draw a circle or oval, use the built-in `ellipse` function with four parameters:

ellipse(x, y, width, height);
  • `x`: Horizontal pixel coordinate of circle center.
  • `y`: Vertical pixel coordinate of circle center.
  • `width`: Horizontal width of the shape in pixels.
  • `height`: Vertical height of the shape in pixels.

// Center Circle Example

function draw() {
  background(30);
  // Draw circle at X=200, Y=200, size 80x80
  ellipse(200, 200, 80, 80);
}

Drawing Boxes: rect()

Drawing rectangles is similar, but they anchor from their **top-left corner** instead of the center.

rect(x, y, width, height, [radius]);
  • `x`: Horizontal pixel position of the top-left corner.
  • `y`: Vertical pixel position of the top-left corner.
  • `width`: Horizontal width of the box.
  • `height`: Vertical height of the box.
  • `[radius]`: (Optional) Curvature of the corners.

// Rectangle with Rounded Corners

function draw() {
  background(30);
  // Box at X=150, Y=100, size 120x80, corner radius 10
  rect(150, 100, 120, 80, 10);
}

Coloring: RGB Values

p5.js shapes are filled using Red, Green, and Blue color parameters (values scale from **0 to 255**).

fill(red, green, blue);
  • `fill(255, 0, 0)`: Bright Red.
  • `fill(0, 255, 0)`: Vibrant Green.
  • `fill(0, 0, 255)`: Deep Blue.
  • `fill(255)`: White (Single value scales from 0/black to 255/white).

Styling Tip:

To remove the outline borders on shapes, call `noStroke()` before drawing them. To set border thickness, use `stroke(r, g, b)` and `strokeWeight(pixels)`.

Step 1: Canvas Setup

We will scale the drawing canvas to fit the full browser window and set a slate dark background.

  • `windowWidth` & `windowHeight` are built-in variables containing the browser frame dimensions.
  • We write background once in `setup()` to clear the screen at initialization.

// Type this in your editor:

function setup() {
  // Fit canvas to window dimensions
  createCanvas(windowWidth, windowHeight);
  // Set slate blue background
  background(15, 23, 42);
}
function draw() {
  // Empty for now
}

Step 2: Cursor Tracker

To draw shapes right at the mouse tip, we replace absolute coordinate values with system variables.

  • `mouseX`: Stores current horizontal cursor coordinate.
  • `mouseY`: Stores current vertical cursor coordinate.
  • These variables update automatically on every frame.

// Add code to the draw function:

function draw() {
  // Clear background with solid color
  background(15, 23, 42);
  
  // Draw circle tracking mouse cursor
  ellipse(mouseX, mouseY, 40, 40);
}

Step 3: The Trail Effect

Why does our circle not leave a line? Because we clear the canvas every frame using `background()` inside `draw()`!

  • Remove `background(15, 23, 42)` from `draw()`.
  • Now background only runs once inside `setup()`.
  • Each circle stacks on top of the previous drawing, leaving a brush line trail!

// Remove background from draw():

function draw() {
  // Background is gone! Circles stack up
  ellipse(mouseX, mouseY, 40, 40);
}

Step 4: Random Values

To make our brush look dynamic and creative, we will generate unpredictable numeric offsets.

  • `random(min, max)`: Selects a random decimal between two bounds.
  • `let`: Declares a local variable placeholder to store the temporary values.

// Declare color variables inside draw():

function draw() {
  // Choose random color components
  let r = random(100, 255);
  let g = random(150, 255);
  let b = random(200, 255);
  
  // ellipse tracking follows next...
}

Step 5: Dynamic Shading

Now we connect our random variables to our shape's color properties.

  • Pass `r`, `g`, and `b` variables directly into `fill()`.
  • Call `noStroke()` to remove shape borders for a cleaner neon look.

// Add color styling commands:

function draw() {
  let r = random(100, 255);
  let g = random(150, 255);
  let b = random(200, 255);
  
  fill(r, g, b);
  noStroke();
  ellipse(mouseX, mouseY, 40, 40);
}

Step 6: Alpha Blending

To create a glowing "neon" trail, we add a fourth parameter to `fill()` to control transparency (Alpha).

  • Alpha scale goes from `0` (invisible) to `255` (completely solid).
  • Let's use `120` to create a semi-transparent, overlapping blending effect.

// Change the fill opacity:

function draw() {
  let r = random(100, 255);
  let g = random(150, 255);
  let b = random(200, 255);
  
  // 120 sets opacity (transparency trail)
  fill(r, g, b, 120);
  noStroke();
  ellipse(mouseX, mouseY, 40, 40);
}

Step 7: Conditional Logic

What if we make a mistake and want to erase the canvas? We can clear the board using an `if` conditional statement.

  • An `if` block checks if a condition is true before running code inside its curly brackets `{}`.
  • `mouseIsPressed`: A built-in system variable that evaluates to `true` when clicked.

Conditional Syntax:

if (condition) {
  // code runs here if true
}

Step 8: Wiping the Canvas

Combine the conditional checks to paint the background slate blue again when the user clicks the mouse.

  • Type this at the very beginning of the `draw()` loop.
  • This clears away old trails and resets the canvas cleanly.

// Add the clear canvas action to draw():

function draw() {
  // Clear canvas if mouse is clicked
  if (mouseIsPressed === true) {
    background(15, 23, 42);
  }
  
  let r = random(100, 255);
  let g = random(150, 255);
  let b = random(200, 255);
  fill(r, g, b, 120);
  noStroke();
  ellipse(mouseX, mouseY, 40, 40);
}

Level 2: Speed-Based Sizing

Can we make the brush size dynamic depending on how fast the user moves the cursor?

  • `pmouseX` & `pmouseY`: Built-in variables containing mouse coordinate positions from the *previous* frame.
  • `dist(x1, y1, x2, y2)`: Calculates distance between two coordinates.
  • This distance represents cursor velocity (speed)!

// Velocity Brush Sizing:

function draw() {
  // Calculate speed as spatial distance
  let speed = dist(pmouseX, pmouseY, mouseX, mouseY);
  
  fill(0, 255, 200, 120);
  // Size matches speed! Fast = Big, Slow = Small
  ellipse(mouseX, mouseY, speed, speed);
}

Game Collision Logic

To catch the falling bubble, we use conditional ranges to verify coordinate overlaps:

if (ballY + ballSize/2 >= catcherY && ballY - ballSize/2 <= catcherY + catcherH) {
  if (ballX >= catcherX - catcherW/2 && ballX <= catcherX + catcherW/2) {
    score = score + 1;
    resetBall();
  }
}
  • Checks if the ball vertical bounding box crosses the paddle.
  • Checks if the ball horizontal alignment lies within the paddle width.

Physics Scaling Fix:

To make sure the ball speed scales with higher scores without breaking at negative values, we capped it with `Math.max(0, score)` in the reset script.
